## Pipeline shape

1. **Checkout** your repository (and any shared contract repos).  
2. **Generate `signals.json`** from tests, eval exports, or monitoring snapshots.  
3. **`geval validate-contract`** on each contract YAML you care about.  
4. **`geval check`** with one or more `--contract` flags.  
5. **Gate on exit code** — `0` merge OK; `1`/`2` per your policy.

## Sketch (pseudo-YAML)

```yaml
steps:
  - name: Export signals
    run: python scripts/build_signals.py > signals.json

  - name: Validate contracts
    run: geval validate-contract .geval/contract.yaml

  - name: Geval check
    run: geval check --contract .geval/contract.yaml --signals signals.json
```
