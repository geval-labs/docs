title: GitHub Actions
description: Run the Geval binary in GitHub Actions — download release, generate signals, gate merges with exit codes.
---

Geval integrates with GitHub Actions the same way as any CLI: **install binary → run `geval check` → interpret exit code**. There is **no** official npm package for the open-source Geval engine described in this documentation.

## Minimal pattern

1. Check out the repo.  
2. Produce **`signals.json`** (your script, eval export, or artifact download).  
3. Download **`geval`** from [GitHub Releases](https://github.com/geval-labs/geval/releases).  
4. Run **`geval check`** with your **`contract.yaml`** and **`signals.json`**.  
5. Use exit codes **`0` / `1` / `2`** for PASS / REQUIRE_APPROVAL / BLOCK.

## Example: released binary on `ubuntu-latest`

```yaml
name: Geval gate

on:
  pull_request:
    branches: [main]

jobs:
  geval:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Geval
        run: |
          curl -fsSL https://github.com/geval-labs/geval/releases/latest/download/geval-linux-x86_64 -o geval
          chmod +x geval

      - name: Build signals
        run: |
          # Replace with your pipeline: Python, Node, or artifact from a prior job
          python scripts/export_signals.py > signals.json

      - name: Run Geval
        run: |
          ./geval check \
            --contract .geval/contract.yaml \
            --signals signals.json \
            --env prod
```

## Multiple contracts

Repeat **`--contract`** for each gate YAML:

```bash
./geval check \
  -c .geval/contract.yaml \
  -c other-team/contract.yaml \
  --signals signals.json
```

Optional: **`--combine-contracts`** (default `worst_case`) controls how overall outcome merges across contracts. See [check](/cli/check).

## Optional: explain on PR

```bash
./geval explain -c .geval/contract.yaml --signals signals.json >> "$GITHUB_STEP_SUMMARY"
```

## Artifacts

`geval check` writes decision JSON under **`.geval/decisions/`** — upload as a workflow artifact if you want historical audit in CI. See [Decision artifacts](/reference/decision-artifacts).
